# Lesson 0 — What is GitHub, and why do we use it?

## The idea in one sentence

GitHub is a place where code lives online, and Git is the tool that tracks
every change to it — like "track changes" in a Word doc, but built for code
and built for teams.

## Why developers use it

- **Backup** — your code lives online, not just on one laptop.
- **History** — every change is saved, so nothing is ever truly lost.
- **Teamwork** — multiple people can work on the same project without
  overwriting each other.
- **Portfolio** — your GitHub profile is basically a public resume of
  everything you've built. Employers look at it.

## Words you'll hear a lot

- **Repository ("repo")** — a project's folder, tracked by Git.
- **Commit** — a saved snapshot of changes, with a short message describing
  what changed.
- **Push** — sending your commits up to GitHub.
- **Pull** — downloading the latest changes from GitHub down to your computer.
- **Clone** — downloading a whole repo for the first time.
