# Lesson 2 — Markdown & README files

You're reading a Markdown file right now! Every `.md` file in this repo is
Markdown — a simple way to format text without needing a fancy word
processor.

## The basics

| You type          | You get                  |
|--------------------|--------------------------|
| `# Big heading`     | # Big heading            |
| `## Smaller heading`| ## Smaller heading       |
| `**bold**`          | **bold**                 |
| `*italic*`          | *italic*                 |
| `- item`            | a bullet point           |
| `` `code` ``        | `code` (monospace)       |
| `[text](url)`       | a clickable link         |

## Why it matters

Every project on GitHub has a `README.md` — it's the first thing anyone sees
when they open a repo. A clear README is one of the easiest ways to look
professional. You'll write your own later when you build your resume site.

## Your task

1. Create a new file: `docs/scratch/your-name.md` (make the `scratch` folder
   if it doesn't exist).
2. Practice: write a heading with your name, a bold sentence about what you
   want to learn this summer, and a bulleted list of 3 things you're
   interested in.
3. Open the preview (magnifying glass icon) to see it rendered.
4. Save, then commit and push it (ask your mentor to walk through this the
   first time — Source Control icon → type a message → checkmark to commit →
   sync icon to push).
